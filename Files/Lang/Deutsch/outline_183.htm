SysInfoSet                                                                                              Funktion

SYSINFOSET(characteristic as string, value as integer)

 

Beschreibung

Setzt den Wert einer System- Charakteristik.

 

 

 

default

CoordinateSystem

Für LoRes auf 72 setzen, auf 144 für HiRes. Siehe Abschneitt 10.J

 

HeapVariableCount

Setzt die Anzahl der numerischen Variablen, für die im Dynamischen Speicherbereich Platz reserviert werden soll. (Default ist 200.) Die Programmausführung wird dadurch beschleunigt, da der Zugriff auf die Variablen schneller erfolgen kann. Variablen werden diesem Bereich in ihrer Reihenfolge bei der Kompilierung zugeordnet. Jede Variable benötigt 22 Byte des dynamischen Speichers. Informieren Sie sich in der Dokumentation des Herstellers, um herauszufinden, wie viel dynamischer Speicher in Ihrem Gerät zur Verfügung steht. Es wird jeweils nur die tatsächlich genutzte Anzahl von Variablen zugeordnet. Die volle Anzahl des  HeapVariableCount wird nur dann benutzt, wenn Sie auch tatsächlich diese Anzahl von Variablen verwenden. Diese Funktion sollte in Ihrem Programm Startup Code implementiert werden.

200

Timer

Die Timer Funktion erlaubt es Ihnen zeitgesteuert, eigene Events auszulösen. Dies ist hilfreich in Situationen, in den Sie innnerhalb eines bestimmten Zeitraumes eine Antwort erwarten und reagieren möchten, falls die Antwort ausbleibt. Sie könnten zum Beispiel bei der Kommunikation mit anderen Computern von Zeit zu Zeit testen, ob irgendwelche Daten angekommen sind. Zum Setzen des Timers nutzen Sie folgenden Befehl:

 

SysInfoSet("Timer", 3000)   'Set a timer in 3 seconds.

 

Zum Auswerten Ihres Timer Events innerhalb des Programms, sollten Sie in Ihrem Event Script etwa so vorgehen:

 

If getEevent()=nsbTimer then

  MsgBox "Timer Event Received" 'respond to event

  SysInfoSet("Timer", 3000)     'set another timer

End If

 

Einen gerade aktiven Timer beenden Sie mit SysInfoSet(“Timer”, 0).Beim Verlassen des Programms wird der Timer beendet. Für dieses Event muss nicht SETEVENTHANDLED genutzt werden. Verwenden Sie zunächst das Timer Beispiel für eigene Versuche.

 

 

 

 

Beispiel

SysInfoSet(“HeapVariableCount”,200)
