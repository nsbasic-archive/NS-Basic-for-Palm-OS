Dim                                                                                                                Befehl

 

DIM varname AS type

DIM varArray(nnn) AS type

DIM varname AS type*length [,decimalPlaces]

DIM varArray(nnn) AS type*length[,decimalPlaces]

DIM varName AS DATABASE dbName, dbRec, dbLayout [,key]

 

Beschreibung

Definiert eine Variable um diese in einer Funktion oder Prozedur zu verwenden. Sofern die Variable eine Liste ist, wird mittels nnn die Anzahl zu erzeugender Listenelemente angegeben.

 

Typ

 

Byte

Ein einzelnes Zeichen. Bei Datenbanklese- und Schreiboperationen wird nur das eine Zeichen geschrieben, ohne das beendende NULL Zeichen.

Database

Definiert eine Datenbank Referenz, auf die Datenbankkommandos angewendet werden können. Wenn die Datenbank an ein Raster (Grid) gebunden werden soll, werden weitere Argumente benötigt.

Date

Definiert ein Datumsformat. Dies ist nicht das Palm-interne Format. Datenbank E/A resultieren in einer 64-Bit langen Fließkommazahl. Das Datum wird intern gespeichert als (Jahr-1900) * 10000 + Monat * 100 + Tag

Double

In allen Aspekten mit Float identisch.

Float

Definiert eine Fließkommazahl, die beides, Ganzzahl und Nachkommaanteil haben kann. Bei Datenbank E/A Operationen wird eine 64-Bit (=8 Byte) doppelgenauigkeits- Fließkommazahl verwendet.

Integer

Definiert eine Ganzzahl ohne Nachkommaanteil. Bei Datenbank E/A Operationen wird eine 32-Bit (4 Byte) Ganzzahl verwendet.

Short

Ähnlich wie Integer, nur dass bei Datenbank Operationen 16-Bit (2 Byte) verwendet wird.

Single

Identisch wie Float, nur dass bei Datenbank Operationen 32-Bit (4 Byte) einfachgenauigkeits- Fließkommazahlen verwendet werden.

String

Zeichenkette mit einer maximalen Länge von 32767 Zeichen.

Time

Definiert einen Zeitwert. Dies ist nicht das Palm-interne Format. Bei Datenbank Operationen wird eine 64-Bit Fließkommazahl verwendet. Eine Zeit wird intern gespeichert als

Stunde * 10000 + Minute * 100 + Sekunde

UserType

Siehe TYPE und END TYPE Anweisung

Variant

Siehe SysTrapFunc und SysTrapSub Funktionen bzw. Prozeduren

 

Length definiert, wie viele Stellen angezeigt werden. DecimalPlaces ist die Anzahle der Dezimalstellen ab dem Dezimalpunkt. Sie muss mindestens 2 Stellen kleiner sein als length.

 

Wenn Sie eine Datenbank zur Verwendung in einem Grid deklarieren, bezeichnet varName den Namen der Variablen. dbName steht für den Namen der Datenbank auf dem Palm OS Gerät. Mit dbRec wird eine globale Variable erzeugt, die den Inhalt des aktuellen Datensatzes der Datenbank enthält. Sie sollte noch nicht dimensioniert sein. dbLayout stellt den Datensatzaufbau dar: er sollte bereits per TYPE Anweisung festgelegt worden sein. key ist eine optionale Variable, die bereits definiert sein muss: es ist der Schlüssel (das Schlüsselfeld) der Datenbank dbName. In dieser Form wird varName nur definiert, wenn sie als erstes Argument einer BindToDataBase Methode für ein Grid verwendet wird.

 

Beispiel

Dim income as float*12,2

Dim count as integer

Dim yesterday as date

Dim appt as time

Dim rates(12) as float

Dim Orders as database

 

Type dbBluesLayout

    Name as String

    age as Integer

    active as Integer

End Type

 

Dim blueKey as Integer

Dim dbBlues1 as Database "Blues" Creator "Grid" Keyed With Record BlueKey, dbBluesRec as dbBluesLayout
